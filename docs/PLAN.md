# Roadmap

Проект развивался поэтапно, каждый релиз добавлял отдельный законченный слой.

| Версия | Этап                                                          |
|--------|---------------------------------------------------------------|
| v0.1.0 | Каркас проекта: конфигурация, логирование, исключения         |
| v0.2.0 | Слой данных: модели PostgreSQL (SQLAlchemy) + клиент Redis     |
| v0.3.0 | Telegram-бот: диспетчер и базовые команды                     |
| v0.4.0 | Движок модерации: ссылки, мат, реклама, спам, флуд            |
| v0.5.0 | ML pipeline: обучение и инференс токсичности                  |
| v0.6.0 | Действия: предупреждения, муты, баны, репутация, роли         |
| v0.7.0 | REST API на FastAPI + JWT-авторизация                         |
| v0.8.0 | Web Dashboard                                                 |
| v0.9.0 | Docker, docker-compose, CI/CD на GitHub Actions               |
| v1.0.0 | Мониторинг (Prometheus/Grafana), финальная документация       |

## Идеи для развития

- Обученная модель токсичности в комплекте + model card с метриками.
- Реальная привязка сервисов (warn/mute/ban) к хендлерам бота через сессию БД.
- Миграции схемы через Alembic вместо `create_all`.
- Расширение датасета и переход на трансформерную модель.

## Требования ТЗ

Сквозные требования соблюдаются на всех этапах: SOLID, типизация, покрытие
тестами не ниже 80%, конфигурация через `.env`, логирование, обработка
исключений и асинхронность там, где это уместно.
